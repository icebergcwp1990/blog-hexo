Python源码剖析-读书笔记

2017.11.14 iceberg 专业

Python源码版本：2.7.14

###Python对象初探###
Python底层实现是基于ANSI C，一般来说，因为创建Python对象是在堆上申请内存即为指针类型，所以是不能静态初始化的。唯一例外的Python的类型对象，Python所有的内建的类型对象都是被静态初始化的。

疑问：1、所谓的静态初始化是否是指在栈上创建对象？2、Python内建的类型对象的特殊性是否意味着它的内存是定长的？

####1.1 Python对象####

  • Python中所有的东西都对象，对应一个PyObject结构体，所有对象都有相同的头部，意味着Python *指针可以引用任意的Python对象
  • 定长对象意味着其对象的值不会影响内存大小，如整数对象等
  • 变长对象通常是容器类型，用PyVarObject【PyObject的拓展】结构体表示，如string对象、list对象等。

####1.2 类型对象####

  • Python对象的元信息存储在PyVarObject中，处理PyObject头之外还包括内存空间大小等信息
  • 创建一个Python对象有两种方法：Python C API和类型对象，如PyInt_Type
  • Python C API分成两类，一类称为范型的API，或者称为AOL。这类API都具有诸如PyObject_*的形式，可以用在任何Python对象上；另一类与类型相关的API，或者称为COL，每一个Python内建对象都有一组对应的类型API。
  • 对于用户自定义类型，Python不可能提供类型相关的API，Python会通过自定义类型对应的类型对象创建实例，类似于Objective-C会创建一颗继承树。
  • 对象的行为有PyTypeObject决定，其中包括了三个非常重要的操作簇：tp_as_number、tp_as_sequence、tp_as_mapping.一种类型可同时指定多种不同对象的行为特征。
  • PyType_Type即类型的类型，是所有class的class,称之为metaclass。
  • 一般对象通过与其关联的类型对象确定其类型，类型对象则通过PyType_Type确定。

####1.3 Python对象的多态性####

  • Python创建的一个PyIntObject对象，内部会用一个PyObject *指针而非PyIntObject *指针保存和维护这个对象，其他对象【包括类型对象】都与此类似。
  • 在Python内部传递的都是范型指针：Python *，指针所指对象具体类型由对象的ob_type域决定。这也是Python多态机制的实现基础。
  • AOL的C API即是建立在多态机制上的，示例如下：
[object.c]
long PyObject_Hash(PyObject *v) {
    PyTypeObject *tp = v->ob_type;
    if (tp->ty_hash != NULL)
        return (*tp->tp_hash)(v);//->优先级高于*
    ....
}

####1.4 引用计数####

  • PyObject中的ob_refcnt是一个32位的整形变量，意味着一个对象的引用不会超过2^32-1。
  • 类型对象不受引用计数规则限制，永远不会被析构，每个对象中指向类型对象的指针不被视为引用。
  • 每一个对象创建的时候，Python提供一个_Py_NewReference(op)宏初始化对象宏为1。
  • 出于效率考虑，Python中采用了内存对象池计数，当一个对象的引用计数减为0时，调用对象的析构函数，但是不意味着一定会调用free释放内存空间。

####1.5 Python对象的分类####
Python对象从概念上大致分为5类

  • Fundamental对象：类型对象
  • Numeric对象：数值对象
  • Sequence对象：序列集合对象
  • Mapping对象：类似于C++中map的关联对象
  • Internal对象：Python虚拟机在运行时内部使用的对象
